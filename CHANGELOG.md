# 更新记录

## v0.3.2

- 紧急修复 v0.3.1 发布包遗漏 `lib/weread.lua`，导致插件启动时无法加载、菜单不显示的问题。
- 恢复完整安装包中的 `NotoEmoji-Regular.ttf`，避免想法弹窗中的表情字体缺失。
- 保留 v0.3.1 的账号清理与设置备份清理修复。
- 增加发布前依赖完整性检查，确认所有 `require("lib.*")` 模块均包含在压缩包中。

## v0.3.1

- 修复“清除账号数据”后可能从 `weread.lua.old` / `weread.lua.new` 恢复登录的问题。
- 清除账号时同步删除插件目录中的 `config.lua` 及常见备份文件，防止下次启动重新导入旧凭证。
- 发布包和 OTA 包均不包含用户设置、Cookie、API Key、Token、运行缓存或本地凭证文件。

## v0.3.0（2026-07-10）

### 新增

- 增加插件内微信读书扫码登录。用户可直接在 KOReader 中显示二维码，使用微信扫码并在手机端确认登录。
- 登录成功后自动验证账号信息，并保存 Cookie 与 API Key，无需手动复制登录凭证。
- 支持扫码登录过程中的验证码输入、取消与错误提示。

### OTA 更新

- 将 OTA 临时文件和备份目录移出 `koreader/plugins`，避免备份目录被误识别为插件。
- 更新时保留用户的 `config.lua`，并避免覆盖 `config.example.lua`。
- 新版本成功启动后再清理 OTA 备份，降低更新失败时无法恢复的风险。
- 自动处理旧版本遗留的 `weread.koplugin.bak-*` 备份目录。
- 增加安全的 `delete_list` 处理，并让安装后版本检查识别统一的 `PLUGIN_VERSION` 常量。

### 修复与调整

- 将 `main.lua`、`_meta.lua`、关于页面和 OTA 清单中的版本号统一为 `0.3.0`。
- 修复此前插件信息页仍显示旧版本号的问题。
- 保留修改版现有的划线/想法显示、想法弹窗、重新下载入口和手动 OTA 检查功能。
