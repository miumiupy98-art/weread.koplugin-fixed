# 更新记录

本文仅记录本增强 Fork 的主要变化。上游项目的历史变化请查看上游仓库及 Git 提交记录。

## [Unreleased]

### Planned

- 增加自动化发布包完整性与敏感信息检查。

## [0.3.5] - 2026-07-13

### Formal release

- OTA 更新清单切换至正式仓库 `miumiupy98-art/weread.koplugin-fixed`。
- 根据评论引用文字重新定位部分失效的 HTML range，改善评论星号、划线与正文错位。
- 保留原始 range 作为评论缓存查询键，避免位置修正后弹窗内容失配。
- 兼容 `abstract` / `contextAbstract`、HTML 实体、换行空格和跨标签文本。

### Integrated from upstream 0.2.1

- 增加书架排序工具栏、阅读状态筛选、下载状态筛选和已缓存标记。
- 书籍详情会校验缓存文件是否真实存在，清除缓存后立即刷新书架状态。
- 整书下载改为章节、划线、想法、图片和 EPUB 打包分阶段执行。
- 单章下载失败时跳过该章并继续；想法按批次下载，失败后最多重试两次。
- 下载进度窗口显示当前阶段，完成后报告失败章节和失败想法批次。
- 使用 KOReader 自带归档器生成 EPUB，`mimetype` 保持不压缩，其余内容使用压缩。
- 修复内容解码索引边界、章节 HTTP 状态检查和文本书格式误判。
- 跨域重定向时移除 Cookie、Authorization 和 Origin；外部资源请求不再附带微信读书 Cookie。
- 在文档初始加载阶段应用划线隐藏设置，降低重复重排风险。

### Preserved from enhanced 0.3.5

- 保留扫码登录、自动初始化阅读上报会话和阅读时间上传修复。
- 保留干净 EPUB / 带划线想法 EPUB 双下载入口。
- 保留想法独立 JSON 缓存与自定义弹窗，不将想法正文写入分页。
- 保留原书脚注处理、OTA 更新和账号安全清理。

## [0.3.3.2] - 2026-07-12

### Personal test build

- 移除正式/个人双通道菜单；本测试构建仅使用个人测试仓库更新地址。
- 将 `succ == 1` 但缺少 `synckey` 的响应视为“请求已接受但未确认”，不再累计为成功。
- 缺少 `synckey` 时使用当前书籍调用 `chapterInfos` 刷新阅读上下文，并重新生成签名后重试一次。
- 为上下文刷新增加 5 分钟冷却，避免持续请求。

## [0.3.3.1] - 2026-07-12

### Personal test build

- 合并阅读时长在线上报稳定性修复与个人测试更新通道。
- 正式稳定通道仍保留；本设备可在设置中切换至个人测试通道。
- 本版本用于个人设备验证，不面向正式用户发布。

## [0.3.3] - 2026-07-11

### Fixed

- 修复 Lua 中 `succ = 0` 被误判为成功的问题，服务器成功现在必须严格满足 `succ == 1`。
- 修复 Cookie 续期后重放旧时间戳和旧签名请求的问题；续期后会重新生成完整上报参数。
- 修复插件实例或阅读器上下文变化后，上报任务依赖临时书籍 ID 而无法恢复的问题。
- 增加明确的停止原因、跳过原因、连续失败次数和最近阅读活动诊断。

### Changed

- 阅读时间上报固定仅在 Reader 打开书籍时运行，不再允许在文件管理器后台持续上报。
- 支持从 KOReader 自带文件管理器、最近阅读或插件书架打开已下载的微信读书 EPUB 后自动识别并上报。
- 连续 10 分钟无翻页活动后暂停定时任务；再次翻页或恢复阅读时自动启动。
- 设备休眠、关闭文档或切换离开阅读器时立即停止上报。
- 无网络时只跳过当前上报，不尝试刷新 Cookie，也不主动开启 Wi-Fi。

## [0.3.2] - 2026-07-11

### Fixed

- 紧急修复 v0.3.1 发布包遗漏 `lib/weread.lua`，导致插件启动时无法加载、菜单不显示的问题。
- 恢复完整安装包中的 `NotoEmoji-Regular.ttf`，避免想法弹窗中的 Emoji 字体缺失。

### Packaging

- 保留 v0.3.1 的账号清理与设置备份清理修复。
- 增加发布前依赖完整性检查，确认所有 `require("lib.*")` 模块均包含在压缩包中。
- 确认完整包不包含用户账号凭据、运行设置和缓存内容。

## [0.3.1] - 2026-07-11

### Security

- 修复“清除账号数据”后可能从 `weread.lua.old` 或 `weread.lua.new` 恢复登录的问题。
- 清除账号时同步删除插件目录中的 `config.lua` 及常见备份文件，防止下次启动重新导入旧凭据。
- 发布包和 OTA 包均排除用户设置、Cookie、API Key、Token、二维码登录状态、运行缓存和本地凭据文件。

## [0.3.0] - 2026-07-10

### Added

- 增加插件内微信读书扫码登录。
- 支持扫码登录过程中的手机确认、四位验证码输入、取消和错误提示。
- 登录成功后自动验证账号并保存 Cookie、API Key 和账号信息。
- 增加手动 Cookie/cURL 导入作为备用登录入口。

### OTA

- 增加插件内手动检查更新。
- 支持 OTA 包下载和 SHA-256 校验。
- 将 OTA 临时文件和备份目录移出 `koreader/plugins`，避免备份目录被误识别为插件。
- 更新时保留用户 `config.lua`，并避免覆盖 `config.example.lua`。
- 新版本成功启动后再清理 OTA 备份，降低更新失败后无法恢复的风险。
- 自动处理旧版本遗留的 `weread.koplugin.bak-*` 目录。
- 增加受限的 `delete_list` 处理和安装后版本检查。

### Changed

- 将 `main.lua`、`_meta.lua`、关于页面和 OTA 清单中的版本号统一。
- 保留并整理划线/想法显示、自定义想法弹窗、重新下载带标注版本等功能。

## [0.2.x-mod] - Earlier modified branch

### Added

- 增加阅读时显示或隐藏划线和想法。
- 增加当前书为干净版本时的提示和重新下载入口。
- 增加想法独立缓存与自定义弹窗显示。

- 调整“更新当前书划线/想法”为重新下载带标注版本。
- 移除部分与当前实现重复或不再维护的实验性入口。

### Fixed

- 修复部分书籍信息、封面和菜单状态处理异常。
